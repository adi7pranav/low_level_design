# ðŸ”¹ 1. CREATIONAL PATTERNS (5)

> Concerned with **how objects are created**

1. **Singleton**  
   Ensures a class has only one instance.

2. **Factory Method**  
   Lets subclasses decide which class to instantiate.

3. **Abstract Factory**  
   Creates families of related objects without specifying concrete classes.

4. **Builder**  
   Constructs complex objects step-by-step.

5. **Prototype**  
   Creates new objects by cloning existing ones.

---

# ðŸ”¹ 2. STRUCTURAL PATTERNS (7)

> Concerned with **how classes and objects are composed**

6. **Adapter**  
   Makes incompatible interfaces work together.

7. **Bridge**  
   Separates abstraction from implementation.

8. **Composite**  
   Treats individual objects and compositions uniformly.

9. **Decorator**  
   Dynamically adds behavior to objects.

10. **Facade**  
    Provides a simple interface to a complex subsystem.

11. **Flyweight**  
    Shares objects to reduce memory usage.

12. **Proxy**  
    Provides a placeholder or surrogate for another object.

18. **Mediator**  
    Reduces coupling between communicating objects.

    Defines a grammar and interpreter for a language.
